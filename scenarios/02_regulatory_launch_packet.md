# Regulatory Launch Packet Builder

- Domain: regulatory workflows and launch readiness.
- Build prompt: Generate a checklist-driven launch packet from structured inputs and incomplete documents.
- Sample inputs: `sample_data/launch_tasks.csv`.
- MVP requirements: group requirements by owner/status, flag gaps, emit a markdown packet.
- Stretch goals: jurisdiction templates, evidence links, approver workflow.
- Production-grade signals: provenance, versioned packet output, explicit unknowns, human approval gates.
- Codex practice loop: inspect schema, create packet generator, validate with sample data.

After completing this scenario: run tests, summarize the diff, write a demo note, run the retrospective prompt, and decide whether exactly one harness improvement should be promoted.
