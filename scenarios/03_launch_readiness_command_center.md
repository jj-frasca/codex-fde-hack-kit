# Launch Readiness Command Center

- Domain: cross-functional launch operations.
- Build prompt: Build a dashboard or CLI summary that shows launch blockers and next actions.
- Sample inputs: `sample_data/launch_tasks.csv`.
- MVP requirements: status counts, overdue tasks, owner summary, blocker list.
- Stretch goals: dependency graph, daily digest, webhook alerts.
- Production-grade signals: clear escalation rules, stale data warnings, owner accountability.
- Codex practice loop: identify smallest UI or CLI, implement summary, add smoke validation.

After completing this scenario: run tests, summarize the diff, write a demo note, run the retrospective prompt, and decide whether exactly one harness improvement should be promoted.
