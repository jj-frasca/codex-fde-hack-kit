# Problem Owner Discovery

Use during or immediately after a live problem-owner conversation.

Capture:

- user/operator
- current workflow
- pain
- trusted data
- constraints
- desired output
- failure mode
- success criteria
- non-goals
- demo expectation

Turn the conversation into:

```text
Problem owner:
Operator:
Current workflow:
Pain:
Trusted data:
Constraints:
Desired output:
Failure mode:
Success criteria:
Non-goals:
Demo expectation:
Open questions:
```

Keep the output concrete. Separate what the owner said from assumptions. Ask follow-up questions only when they would change the build plan.
