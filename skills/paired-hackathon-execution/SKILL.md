# Paired Hackathon Execution

Use when two candidates or builders are working with one problem owner during a timed hackathon.

Principles:

- align quickly
- split work clearly
- avoid ego
- set sync checkpoints
- keep problem owner involved
- integrate early
- demo one coherent artifact
- avoid building disconnected features

Output:

```text
Shared goal:
Division of labor:
My owner area:
Partner owner area:
Sync checkpoints:
Problem-owner check-ins:
Integration plan:
Demo roles:
Fallback plan:
Risks:
```

Prefer one useful artifact over two impressive but disconnected pieces. If the split creates integration risk, reduce scope before adding features.
