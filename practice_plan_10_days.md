# 10 Day Practice Plan

Day 1: Install the harness, read every prompt, and run repo triage on a familiar repo.

Day 2: Practice worktree creation and removal. Confirm Codex edits stay isolated.

Day 3: Build the site acquisition triage MVP from sample CSV data.

Day 4: Build the launch readiness summary. Focus on blockers and owners.

Day 5: Build community inbox triage with deterministic rules and review states.

Day 6: Build webhook alert formatting from JSONL events.

Day 7: Practice debugging: break a smoke check, ask Codex to reproduce and fix it.

Day 8: Practice review: ask Codex to review its own branch and prioritize findings.

Day 9: Timed drill: 90 minutes from repo inspection to verified diff summary.

Day 10: Demo drill: explain the operational problem, show messy input, show useful output, state limitations.
