# Repo Deep-Dive Report Schema

Use this structure for the final synthesis.

## Understanding

- What the repo appears to do.
- Primary users/operators.
- Main workflows.
- Confidence level and unknowns.

## Architecture

- Main modules and responsibilities.
- Entrypoints: CLI, API, UI, jobs, scripts.
- Data flow from input to output.
- External systems, APIs, databases, files, queues, or model calls.

## Commands

- Install/setup.
- Test/lint/typecheck.
- Run app/CLI.
- Demo/smoke commands.
- Any command uncertainty.

## Key Flaws And Risks

List findings by severity. Include file references when possible.

```text
Severity:
Finding:
Evidence:
Why it matters:
```

## Top One-Day Improvements

Give three options.

```text
Option:
User value:
Likely files:
Validation:
Demo signal:
Risk:
```

## Recommended Slice

- One-sentence scope.
- Non-goals.
- Problem-owner questions.
- Files likely touched.
- Step-by-step plan.
- Focused verification.
- Demo narrative.

## Implementation Guardrails

- What not to edit yet.
- Data/privacy/secrets concerns.
- Dependencies to avoid unless approved.
- Human-review boundaries for GenAI workflows.
