Turn the problem-owner briefing into a buildable plan.

Context:
- This starts after a live problem show.
- Owners outline specific pain points, context, constraints, and desired outcomes.
- The goal is a useful artifact by demo time, not a polished platform.
- Stay concrete and separate facts from assumptions.

Extract:
- problem owner
- current workflow
- pain points
- desired outcome
- trusted inputs
- required outputs
- constraints
- risk if wrong
- what "useful by demo time" means
- smallest useful artifact
- explicit non-goals
- validation path
- demo path
- open questions for the problem owner

Output format:

```text
Problem owner:
Current workflow:
Pain:
Desired outcome:
Trusted inputs:
Useful output:
Constraints:
Risk if wrong:
Smallest useful artifact:
Non-goals:
Validation:
Demo path:
Open questions:
```
