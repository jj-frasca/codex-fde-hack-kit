Using the problem-owner discovery notes, create a concise agent/tool spec suitable for sharing with the evaluating team.

Use this structure:

- Problem Owner
- User / Client
- Current Workflow
- Pain / Bottleneck
- Trusted Inputs
- Desired Output
- Proposed Artifact
- Smallest Useful Version
- Explicit Non-Goals
- Workflow
- Guardrails / Trust
- Implementation Plan
- Validation Plan
- Demo Plan
- Open Questions

Keep it practical, not academic.

Optimize for usefulness, trust, and demo feasibility.
